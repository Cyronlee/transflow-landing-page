# TransFlow v1.4.0 发布：视频转录 + 说话人识别来了

> 导入视频，自动区分「谁说了什么」——TransFlow 最大的一次功能更新。

---

## 这次更新了什么？

v1.4.0 是 TransFlow 自发布以来**信息密度最高的一次更新**，核心围绕一个新场景：

**把视频变成带说话人标注的文字稿。**

---

## 新功能

### 🎬 视频转录

现在你可以直接导入视频文件（mp4、mov 等），TransFlow 会自动提取音频并转录为文字。

不需要额外工具，不需要先转换格式——拖进来就能用。

### 🗣️ 说话人识别（Speaker Diarization）

这是本次更新最核心的能力。

TransFlow 会自动分析音频中的不同声纹，识别出每段话是**谁在说**，并用不同颜色标注。

> 👇 视频转录 + 说话人识别效果

适用场景：

- **会议录像** — 自动区分参会者发言
- **访谈节目** — 主持人和嘉宾一目了然
- **课堂录屏** — 老师和学生的对话自动分开

### 🎙️ 录音保存

转录过程中的录音现在会自动保存到会话文件，方便回放和核对。

---

## 体验优化

### 悬浮预览面板升级

悬浮窗的控制按钮做了全新设计：

- **水平布局** — 字号、置顶、关闭按钮一字排开，浮在文字上方
- **字号调节** — 从原来的加/减按钮改为下拉菜单，支持 12–72pt，一步到位
- **更大的点击区域** — 按钮热区扩大，不再容易误触

> 👇 悬浮预览面板

### 自动滚动开关

转录视图右下角新增自动滚动按钮。默认开启，新内容自动滚到底部；手动翻阅历史内容时可以关闭，不再被打断。

### 全新 Logo

应用图标更新了设计，更加简洁现代。

## Bug 修复

这个版本修复了几个影响稳定性的问题：

| 问题 | 修复 |
|------|------|
| DMG 版本打开历史视频时崩溃 | 替换 SwiftUI VideoPlayer 为原生 AVPlayerView |
| 边转录边翻译时切换 Tab 崩溃 | 将 TranslationSession 提升到更高层级管理 |
| 新建会话时「创建」按钮无法点击 | 修复名称预填充时的状态判断 |
| 内容起始时间计算错误 | 修正时间戳基准点 |

## 接下来的计划

v1.4.0 实现了**视频文件的说话人识别**，但这只是开始。未来版本计划支持：

- **实时说话人识别** — 在麦克风和应用音频场景下也能区分说话人
- **自定义词汇表** — 添加专业术语，提高特定领域的转录准确率
- **后期精校准** — 对转录结果进行二次编辑和修正
- **全局音频监听** — 系统级音频捕获，不限于单个应用
